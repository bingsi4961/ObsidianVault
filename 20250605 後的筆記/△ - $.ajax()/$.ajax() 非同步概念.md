### 1. 為什麼我們需要「非同步」？ (實務上解決什麼問題)

想像一下，如果你在網頁上點擊一個按鈕去讀取資料庫的幾萬筆報表，如果沒有「非同步」機制，你的網頁在等待後端處理的這幾秒鐘內，會呈現完全卡死（凍結）的狀態。你不能滑動頁面、不能點其他按鈕。非同步的出現，就是為了讓網頁在「等待」的過程中，依然能保持流暢的操作體驗。

### 2. 日常生活比喻：去速食店點餐 🍔

要理解非同步，我們可以用「去速食店點餐」來比喻：

- **同步 (Synchronous)：** 你在櫃檯點了一份炸雞。這時你**死死站在櫃檯前不走**，盯著店員看。直到十分鐘後炸雞炸好，你拿到餐點了，才走回座位。這十分鐘內，排在你後面的客人都被你卡住了（網頁凍結）。
    
- **非同步 (Asynchronous)：** 你點了炸雞，店員給你一個「叫號機」。你拿著叫號機先回座位滑手機、跟朋友聊天（繼續執行後續程式碼）。當廚房把炸雞做好的時候，叫號機會逼逼叫，你再去櫃檯把餐點拿回來。    

### 3. 全貌與運作架構 🗺️

在 JavaScript 的世界裡，因為它是「單執行緒」（一次只能做一件事），所以如果遇到要花時間等待的任務（例如 AJAX 呼叫後端），它會把這個任務丟給瀏覽器去背景處理，自己先往下跑。

整體的執行脈絡是這樣的：

1. **發送請求**：執行到 `$.ajax()`，就像跟店員點完餐。
    
2. **繼續前進**：JavaScript **不會停下來等**，而是立刻執行 `$.ajax()` 區塊下方的程式碼（回座位跟朋友聊天）。
    
3. **背景處理**：後端正在努力撈資料（廚房炸雞中）。
    
4. **收到回應並回呼**：後端資料處理完畢並回傳。這時會觸發你事先寫好的 `success` 函數（叫號機響了，去拿餐點）。這個函數我們在專業術語上稱為 **回呼函式 (Callback Function)**。    

### 4. 程式碼範例解析 💻

```javascript
console.log("1. 點餐：開始發送 AJAX 請求");

$.ajax({
  url: "/api/get-data",
  method: "GET",
  success: function(response) {
    // 這裡就是「叫號機響了」才會執行的代碼
    console.log("4. 拿到餐點：後端回應資料是", response);
  }
});

console.log("2. 點完餐先回座位：AJAX 請求已發送，繼續執行其他程式碼");
// 3. 後端在這段時間內處理資料...
```

執行結果的順序永遠會是：`1 -> 2 -> 4`。

這解釋了你的疑問：**`success` 絕對不會在後端處理完之前就執行**。它是被「預先設定好」的機關，只有當資料真的抵達時，才會被觸發。

### 5. 對比：同步 vs 非同步 ⚖️

我們用一個表格來對比兩者：

|**比較項目**|**同步 (Synchronous)**|**非同步 (Asynchronous)**|
|---|---|---|
|**執行順序**|一行一行等，上一行沒做完，下一行不能動|發出耗時請求後，不管結果，直接做下一行|
|**網頁狀態**|等待期間畫面會卡死 (UI 阻塞)|等待期間畫面依然可以操作 (非阻塞)|
|**取得資料的方式**|直接回傳結果給變數|透過 Callback (如 success) 或 Promise 接收|
|**實務應用**|簡單的數學計算、字串處理|網路請求 (AJAX)、讀寫檔案、計時器 (setTimeout)|

### 6. 什麼時候該用？什麼時候不該用？ 🚦

- **該用的時候**：跟伺服器要資料、上傳圖片、任何需要花費未知的時間才能完成的動作，**預設都應該使用非同步**。
    
- **不該用的時候 (或者說何時會改成同步 `async: false`)**：在非常古老的網頁技術中，有時為了在使用者關閉瀏覽器視窗的那一瞬間（`window.onbeforeunload`）硬是把資料傳回後端，會把 AJAX 設為同步。但現在極度不建議這麼做，因為這會導致瀏覽器卡頓，現代寫法會改用 `navigator.sendBeacon`。    

### 7. 常見的錯誤踩坑經驗 💣

這是剛學非同步的人**最常犯的錯誤**，請一定要注意！很多人會試圖把 AJAX 抓到的資料拿出來給外面的變數用：

```javascript
let myData = null;

$.ajax({
  url: "/api/get-data",
  success: function(response) {
    myData = response; // 賦值
  }
});

console.log(myData); // 請問這裡會印出什麼？
```

這時印出來的會是 `null`！為什麼？因為 `console.log(myData)` 在 AJAX 發送出去的瞬間就已經被執行了（這時後端根本還沒回傳，`success` 也還沒執行）。

**避坑心法**：跟 AJAX 資料有關的後續處理，**全部都要寫在 `success` 裡面**，或是把它包裝成另一個 Function 在 `success` 裡面呼叫。

### 8. 小結 📌

非同步不是「不等待結果」，而是「不留在原地死等」。我們把後續要處理資料的邏輯寫在 `success` (回呼函式) 裡，交給瀏覽器在拿到資料的那一刻自動幫我們喚醒並執行。
