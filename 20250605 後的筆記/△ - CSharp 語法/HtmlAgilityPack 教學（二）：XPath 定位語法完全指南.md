## 什麼是 XPath？🧭

XPath 全名是 **XML Path Language**，你可以把它想像成在網頁地圖中的 **GPS 定位系統**。

當我們在寫爬蟲或自動化測試時，HTML 網頁就像一棟充滿房間的大樓，而 XPath 就是精確的指令，告訴程式：「去 3 樓左手邊第二個房間，拿走那個紅色的盒子」。

在學習語法前，我們先認識核心概念：**網頁是一棵「樹」** 🌲。HTML 是由標籤（Tags）組成的層級結構，每個標籤就是這棵樹上的一個節點。而 XPath 就是在這棵樹上「指路」的語言。

### XPath 核心符號速查表

|符號|名稱|說明|比喻|
|---|---|---|---|
|`/`|單斜線|從根目錄開始，或找「直接下一層」|親生孩子 👨‍👩‍👦|
|`//`|雙斜線|在整份文件中尋找，不論層級多深|所有的後代子孫 👵|
|`[]`|中括號|加入篩選條件|穿紅衣服的孩子 👕|
|`@`|艾特符號|選取標籤內的屬性（如 id, class）|孩子的身分證字號 🆔|

接下來我們將從最基礎的標籤定位開始，一步步帶你掌握完整的 XPath 語法。

---

## 一、基礎標籤與屬性定位 🎯

在 HTML 中，每個元素都有自己的**標籤名稱**（如 `<div>`、`<a>`），大多數重要的元素還會有**屬性**（如 `id` 或 `class`），就像是它們的身分證或特徵。

### 用標籤名稱定位

最簡單的 XPath 就是直接寫標籤名。例如 `//h1` 會找出頁面上所有的一級標題。

這就像在廣播說：「請所有**姓 h1** 的人過來。」

### 用屬性篩選定位

使用 `[]` 中括號配合 `@` 符號，可以進一步縮小範圍：

```
//div[@id='main']       → 找 ID 為 main 的 div 🆔（ID 通常是唯一的）
//a[@class='link']      → 找類別（Class）為 link 的超連結 🏷️
```

這就像改成說：「請那位**姓 p** 且**身分證號碼是 intro** 的人過來。」——範圍立刻精確到只剩一個人。

### 練習範例

假設我們正在爬取一個購物網站，網頁原始碼片段如下：

```html
<div class="product-list">
    <h2 id="promo-title">今日特惠</h2>
    <a href="/p123" class="buy-btn">立即購買</a>
    <a href="/p456" class="buy-btn">加入購物車</a>
</div>
```

想精確定位「今日特惠」標題，結合標籤名和 ID 屬性：

```
//h2[@id="promo-title"]
```

## 二、路徑層級的奧秘 — `/` 與 `//` 的差異 🌳

這是 XPath 中最核心、也是新手最容易搞混的概念。讓我們用一個清楚的結構來說明它們的差異。

### `/`（單斜線）：只找「親生孩子」👨‍👦

代表 **直接子節點 (Direct Child)**，它要求目標必須緊接著在父標籤的「下一層」。

### `//`（雙斜線）：找「所有子孫」👵👴

代表 **後代節點 (Descendant)**，它會搜尋目標標籤下方的**任何層級**，不論隔多深都找得到。

### 比較表

|符號|意義|動作|
|---|---|---|
|`/`|子節點|往下一層找「親生孩子」|
|`//`|後代節點|往下找「所有子孫」|

### 實際案例對照

假設 HTML 結構如下：

```html
<section id="hot-news">
    <div class="wrapper">
        <div class="item">新聞內容</div>
    </div>
</section>
```

在這個結構中，`<div class="item">` 被包在 `wrapper` 裡面，它是 `section` 的「**孫子**」而不是「兒子」。

|XPath 寫法|能抓到嗎？|原因|
|---|---|---|
|`//section[@id='hot-news']/div[@class='item']`|❌ 不行|`/` 只看下一層，那一層只有 `wrapper`|
|`//section[@id='hot-news']//div[@class='item']`|✅ 可以|`//` 會翻遍所有子孫層級|

路徑展開來看：

- **使用 `/div[@class='item']`：** `section` ➔ （找不到 `item`）❌
- **使用 `//div[@class='item']`：** `section` ➔ `wrapper` ➔ `item` ✅

> 💡 **老師的建議**：在寫爬蟲時，除非你百分之百確定網頁結構永遠不會變，否則使用 `//` 會讓你的程式碼更有**彈性**（Robust），不會因為前端工程師多加了一個包裝用的 `div` 就整台壞掉。

## 三、相對路徑的「句點」(`.`) — 新手最常踩的坑 📍

這是一個看似微小、卻影響極大的細節。

### 問題場景

假設你已經用 `SelectSingleNode` 抓到了一個新聞區塊的節點 `newsNode`，現在你想從裡面找連結：

```csharp
// ⚠️ 錯誤寫法：會從「整份文件」開始找！
var aNode = newsNode.SelectSingleNode("//a");

// ✅ 正確寫法：只在「目前節點」裡面找
var aNode = newsNode.SelectSingleNode(".//a");
```

### 差異說明

- `//a`：在「全家」（整份網頁）找第一個連結。
- `.//a`：只在「目前這個房間」（`newsNode`）裡面找第一個連結。

如果你寫 `newsNode.SelectSingleNode("//a")`，HAP 會從**整份文件的最開頭**開始找，結果你可能會抓到網頁最上方的導覽列連結，而不是這則新聞裡面的連結。

這就像你在找「冰箱裡的鮮奶」，如果你沒說「冰箱裡」，機器人可能會跑到超市去拿一罐給你。😂

> ⚠️ **記住這條黃金規則**：只要你是在某個子節點上做搜尋，XPath 開頭一定要加一個**點（`.`）**，才能確保搜尋範圍被限制在該節點之內。

## 四、精準打擊 — 用 ID 與 Class 縮小範圍 🎯

在實際的網頁中，同名的標籤通常多到數不清（例如廣告、大頭貼、背景圖）。如果你只想抓到「特定的那一個」，我們通常會利用 HTML 的 **`class`** 或 **`id`** 屬性。

想像網頁原始碼長這樣：

```html
<img src="ads.jpg" class="ad-banner">
<img src="avatar.jpg" class="user-photo">
<img id="main-logo" src="logo.png">
```

你可以把 XPath 寫得更精準：

```
根據 ID 定位：//*[@id="main-logo"]
根據 Class 定位：//img[@class="user-photo"]
```

這裡的 `[@屬性名="值"]` 就像是給機器人一個更詳細的「門牌號碼」，讓它不會搬錯箱子。

> 💡 **小提醒**：`//*[@id="main-logo"]` 中的 `*` 是萬用字元，代表「任何標籤名稱」。當你確定 ID 是唯一的，不需要指定標籤類型時，`*` 就很好用。

## 五、處理多個類別 — `contains()` 函數 🏷️

在實務中，`class` 屬性通常比 `id` 更常遇到，但它有個小陷阱：**一個標籤可能同時擁有多個類別**。

例如：`<a class="buy-btn primary active">`

如果你寫 `//a[@class='buy-btn']`，XPath 會因為**不是完全匹配**（整個 class 值是 `"buy-btn primary active"`，不等於 `"buy-btn"`）而找不到它！

這時候我們需要 `contains()` 函數：

```
語法：//標籤[contains(@屬性, '部分文字')]
例子：//a[contains(@class, 'buy-btn')]
```

這就像在說：「幫我找所有的連結，只要它的類別裡面**包含**『buy-btn』這幾個字就好。」

> ⚠️ **注意**：`[@class='buy-btn']` 是「完全匹配」，整個 class 值必須一模一樣才行。而 `[contains(@class, 'buy-btn')]` 是「部分匹配」，只要包含就算。在現代網頁中（一個標籤通常有三、四個類別），幾乎都會使用 `contains()` 來處理 class。

---

## 六、根據文字內容定位 🗣️

有時候，標籤沒有 `id` 也沒有獨特的 `class`，但它顯示的**文字**是固定的。這時候我們可以用 `text()` 函數。

### 精確匹配

```
//a[text()='點我閱讀']
```

這會找到文字**剛好就是**「點我閱讀」的連結。

### 部分匹配

```
//a[contains(text(), '閱讀')]
```

這會找到文字中**包含**「閱讀」兩個字的連結（例如：「點我閱讀」、「繼續閱讀」都會被抓到）。

這在爬取按鈕（例如「下一頁」、「搜尋」）或特定標題時非常強大。

### 練習範例

假設網頁上有一個按鈕：`<button class="btn-style">提交表單</button>`

如果你想透過「提交」這兩個關鍵字來定位這個按鈕：

```
//button[contains(text(),'提交')]
```

### 進階細節：`text()` vs `.`（點）

在實務中，有個細節常讓工程師卡住：如果文字被包在更深層的標籤裡怎麼辦？

例如：`<button><span>提交</span>表單</button>`

- **使用 `text()`**：它只看 `button` 直接夾住的文字。在這個例子中，`button` 的直接文字只有「表單」二字（「提交」在 `span` 裡），所以 `//button[text()='提交表單']` 會定位失敗。❌
- **使用 `.`（點）**：`//button[contains(., '提交')]`。這個「點」代表該節點的所有**字串值 (String-value)**，它會把裡面所有子孫標籤的文字通通「揉成一團」來檢查。✅

## 七、組合多重條件 — AND / OR 🧩

在實務中，網頁標籤往往很「大眾臉」。例如，頁面上可能有 10 個 `class="btn"` 的按鈕。如果你只想抓那個「藍色」且文字是「確認」的按鈕，我們就需要組合多個條件。

### `and`（且）— 兩個條件都要符合

```
語法：//標籤[@屬性1='值' and @屬性2='值']
例子：//button[@class='btn-blue' and text()='確認']
```

### `or`（或）— 符合其中一個即可

```
語法：//標籤[@屬性1='值' or @屬性2='值']
```

### 練習範例

假設 HTML 如下：

```html
<div class="container">
    <button class="menu-item active">首頁</button>
    <button class="menu-item">產品介紹</button>
    <button class="menu-item active">聯絡我們</button>
</div>
```

想精確定位「正在使用中 (active)」且文字是「首頁」的按鈕：

```
//button[contains(@class,'active') and text()='首頁']
```

一個條件是 `@class` 要包含 `active`，另一個條件是 `text()` 要等於 `首頁`，兩者都滿足才會被選中。

## 八、位置與索引 — 精準點名 🔢

有時候，網頁上一整排標籤的屬性完全一模一樣（例如：一排新聞清單或商品列表）。當屬性無法區分時，我們可以用「順序」來點名。

### 常用語法

|目標|語法|說明|
|---|---|---|
|第一個|`[1]`|XPath 索引從 **1** 開始，不是 0！|
|最後一個|`[last()]`|`last()` 是內建函數|
|倒數第二個|`[last() - 1]`|可以做數學運算|
|前兩個|`[position() < 3]`|`position()` 回傳當前位置|

> ⚠️ **注意**：XPath 的索引是從 **1** 開始，這跟大多數程式語言（從 0 開始）不同，千萬別搞混了！

### 關於 `last()` 的用法

`last()` 是一個**函數**，它會回傳目前這組結果的「總數量」（也就是最後一個人的編號）。它**不接受參數**（括號內不能放數字）。

如果你想表達「倒數第幾個」，需要用數學運算：

```
最後一個：    [last()]       → 共 N 個，拿第 N 個
倒數第二個：  [last() - 1]   → 共 N 個，拿第 N-1 個
倒數第三個：  [last() - 2]   → 依此類推
```

### 索引的小陷阱 🪤

如果你寫 `//li[1]`，有時候會抓到「每個清單裡的第一個」（因為每個 `<ul>` 裡面各有一個 `[1]` 的 `<li>`）。如果要抓「整份文件裡的第一個 `<li>`」，通常會用括號包起來：`(//li)[1]`。

### 練習範例

假設有一個熱門文章列表：

```html
<ul class="hot-articles">
    <li>1. AI 革命：未來十年的變化</li>
    <li>2. 寫程式不再痛苦的秘密</li>
    <li>3. 如何成為爬蟲大師</li>
    <li>4. 2026 年技術趨勢報告</li>
</ul>
```

抓第一則：`//ul[@class='hot-articles']/li[1]`

抓最後一則：`//ul[@class='hot-articles']/li[last()]`

抓第二則（手機在三項清單中也是倒數第二個）：`//li[2]` 或 `//li[last() - 1]`（視清單長度而定）

## 九、家族成員導航 — 軸 (Axes) 🧭

這是 XPath 最強大的武器：**當目標標籤本身沒有任何特徵時，透過它的「鄰居」來找到它**。

### 常用的軸

|軸|方向|意義|
|---|---|---|
|`following-sibling::`|同一層，往後找|找右邊的兄弟|
|`preceding-sibling::`|同一層，往前找|找左邊的兄弟|
|`parent::`|往上一層|找爸爸|
|`..`|往上一層|`parent::` 的簡寫|

> ⚠️ **注意拼字**：軸的名稱是**單數** `following-sibling`，不是 `following-siblings`。

### 實戰案例

想像 HTML 長這樣：

```html
<div class="row">
    <span class="label">價格：</span>
    <span class="value">999</span>
</div>
```

如果你想抓那個 `999`，但它的 class `value` 到處都是。你可以先找到文字是「價格：」的標籤，然後找它**右邊的鄰居**：

```
//span[text()='價格：']/following-sibling::span
```

### 如何理解 `/` 在軸導航中的角色？

你可能會困惑：「`/` 不是代表下一層嗎？為什麼 `following-sibling` 是同一層？」

其實在 XPath 的標準定義中，`/` 是一個**「步驟分隔符」**（Step Separator），它代表的是「然後」，而不只是「下一層」。方向取決於 `/` 後面跟著的**軸 (axis)**：

- 如果動作是 `child::`（預設，省略不寫），就會往**下一層**找。
- 如果動作是 `following-sibling::`，就會往**同一層後面**找。
- 如果動作是 `parent::`，就會往**上一層**找。

把它想成 GPS 指令之間的「然後」就好：

1. **第一步**：`//td[text()='香蕉']`（找到文字是香蕉的格子）。
2. **然後**（`/`）...
3. **第二步**：`following-sibling::td`（沿著兄弟軸線往後找）。

### 綜合練習

假設網頁上有一個表格：

```html
<table id="inventory">
    <tr>
        <td>蘋果</td>
        <td>$50</td>
    </tr>
    <tr>
        <td>香蕉</td>
        <td>$30</td>
    </tr>
</table>
```

想精確定位到「香蕉」旁邊的「$30」：

```
//td[text()='香蕉']/following-sibling::td
```

先找到文字是「香蕉」的 `td`，再往同一層後方找它的兄弟 `td` 節點，就能拿到 `$30`。

## 十、`//` 的完整定義 — 解答常見疑惑

在學完軸 (Axes) 的概念之後，我們可以更精確地理解 `//` 到底是什麼了。

在 XPath 的完整語法中，`//` 不能單純解釋成兩個 `/` ，實際上是 `/descendant-or-self::node()/` 的**縮寫**。雖然聽起來很複雜，但簡單來說：

- **單斜線 `/`**：對應 `child::` 軸（兒子軸）。它只會看目前的下一層。
- **雙斜線 `//`**：對應 `descendant-or-self::` 軸。它會從目前位置開始，往下搜尋**所有層級**（包含自己、兒子、孫子、曾孫...）。

所以當你寫 `//div//a` 時，它的意思是：先在文件任何地方找到 `div`，然後在該 `div` 裡面的**任何深度**尋找 `a`。

## XPath 語法總整理 📋

|分類|語法|說明|範例|
|---|---|---|---|
|**標籤定位**|`//標籤`|找任何位置的該標籤|`//h1`|
|**屬性篩選**|`[@屬性='值']`|完全匹配屬性值|`//div[@id='main']`|
|**部分匹配**|`[contains(@屬性, '值')]`|屬性值包含某字串|`//a[contains(@class, 'btn')]`|
|**文字定位**|`[text()='文字']`|精確匹配直接文字|`//a[text()='首頁']`|
|**文字模糊**|`[contains(text(), '文字')]`|文字包含某字串|`//button[contains(text(), '提交')]`|
|**全文比對**|`[contains(., '文字')]`|含子孫的全部文字|`//button[contains(., '提交')]`|
|**組合條件**|`[條件1 and 條件2]`|同時滿足|`//button[contains(@class,'active') and text()='首頁']`|
|**子節點**|`/`|只找下一層|`//div/a`|
|**後代節點**|`//`|找任意深度|`//div//a`|
|**相對路徑**|`.//`|從目前節點往下找|`node.SelectSingleNode(".//a")`|
|**索引**|`[n]`|第 n 個（從 1 開始）|`//li[1]`、`//li[last()]`|
|**兄弟導航**|`following-sibling::`|同層往後找|`//td[text()='香蕉']/following-sibling::td`|
|**父節點**|`..` 或 `parent::`|往上一層|`./..`|
