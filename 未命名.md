## 寫在前面

第 1 篇我們把觀念都搞懂了，這一篇我們要把手弄髒——直接在 SSMS v17.6 的介面上操作。

好消息是：v17.6 的圖形化介面（GUI）非常直覺，而且跟後續新版本的操作位置幾乎一模一樣，所以你學會這個版本，未來升版也不會有適應問題。

**操作前的共通準備**：請在左側的「**物件總管 (Object Explorer)**」中，依序展開：

```
資料庫 → 你的資料庫 → 資料表
```

接下來的所有操作都從這個位置出發。

---
## 一、設定主鍵（Primary Key）

### 操作步驟

1. 在目標資料表上**按右鍵**，選擇「**設計 (Design)**」。
2. 點選你要作為主鍵的資料行（欄位）**最左側的灰色小方塊**。
3. 按右鍵，選擇「**設定主索引鍵 (Set Primary Key)**」。
4. 該欄位旁邊會出現一把**金色的鑰匙圖示** 🗝️。
5. 按下 `Ctrl + S` 儲存資料表，完成！

### 設定複合主鍵（多欄位組成的主鍵）

如果要設定複合主鍵，先按住 `Ctrl` 點選**多個欄位**最左側的小方塊，再按右鍵選擇「設定主索引鍵」。所有被選中的欄位旁邊都會出現金鑰匙圖示。記住，這樣設定出來的依然是「一個」主鍵，只是由多個欄位組成而已。

---
## 二、設定外部索引鍵（Foreign Key）

外部索引鍵是為了建立兩張表之間的關聯，以「子表（含 FK）關聯到父表（含 PK）」為例。

### 操作步驟

1. 在目標資料表（**子表**）上按右鍵，選擇「**設計 (Design)**」。
2. 在設計視窗的空白處（或欄位上）按右鍵，選擇「**關聯性 (Relationships)**」。
3. 點選左下角的「**加入 (Add)**」，產生一個新的關聯。
4. 在右側的屬性視窗中，找到「**資料表及資料行規格 (Tables and Columns Specification)**」，點選該列右側的 `...` 小按鈕。
5. 在跳出的視窗中設定：主索引鍵資料表（父表）和對應欄位，以及外部索引鍵資料表（保持目前的子表）和對應欄位。
6. 按下「確定」並儲存資料表。

### 實務舉例（Portal 系統情境）

假設有 `Orders` 表和 `Customers` 表，要讓 `Orders.CustomerId` 關聯到 `Customers.CustomerId`：

- 子表 = `Orders`（從這張表開始操作）
- 主索引鍵資料表 = `Customers`、欄位 = `CustomerId`
- 外部索引鍵資料表 = `Orders`、欄位 = `CustomerId`

設定完之後，你想 INSERT 一筆 `CustomerId = 999` 但 `Customers` 表沒這個客戶，SQL Server 就會直接擋下來。

### FK 的「方向性」與物件歸屬——這個觀念很重要

> **FK 永遠是「子表 → 父表」的單向參考，而且 FK 這個物件在系統層級是「登記在子表名下」的。**

```
Customers 表（父表）              Orders 表（子表）
  ┌──────────────┐               ┌──────────────────────┐
  │ CustomerId PK│ ←─────────────│ CustomerId FK ← 這個 FK 屬於 Orders
  │ Name         │   「參考」    │ OrderId PK           │
  └──────────────┘               │ OrderAmount          │
                                 └──────────────────────┘
```

這代表：**你去展開 `Customers`（父表）的「索引鍵」節點，不會看到這條 FK**，因為它不屬於 `Customers`。你要展開 `Orders`（子表）的「索引鍵」節點才看得到。

### 想看「誰參考了我的主鍵」怎麼辦？

既然父表的「索引鍵」節點看不到入向 FK，實務上又常需要知道「有哪些表參考了我」（例如想刪除一筆客戶資料前先確認），有三種方法：

#### **方法 1：用「檢視相依性」**

在 `Customers` 表上按右鍵 → **「檢視相依性 (View Dependencies)」** → 選「**相依於 [Customers]**」的物件，可以看到所有參考它的子表。

#### **方法 2：用 T-SQL 查（最精準，記下來！）**

```sql
-- 查詢「哪些表的 FK 指向 Customers 表」
SELECT
    OBJECT_NAME(fk.parent_object_id)     AS 子表,
    OBJECT_NAME(fk.referenced_object_id) AS 父表,
    fk.name                              AS FK名稱
FROM sys.foreign_keys fk
WHERE OBJECT_NAME(fk.referenced_object_id) = 'Customers';
```

這會列出所有「指向 Customers」的 FK 清單，實務上超級實用。

#### **方法 3：用「資料庫圖表 (Database Diagram)」**

在資料庫節點下找到「資料庫圖表」，建立新圖表並將相關資料表拉進來，即可以視覺化方式瀏覽所有關聯。
此外，在圖表中對 FK 連接線按右鍵 → 選擇「屬性 (Properties)」，於「資料表及資料行規格 (Tables and Columns Specification)」區塊中，可以清楚看到：

- **主索引鍵資料表（父表）**：被參考的那一方 
- **外部索引鍵資料表（子表）**：持有 FK 的那一方 
- 以及雙方對應的欄位名稱

#### **方法 4：資料表設計工具 → 關聯性對話框**

1. 在 SSMS 對著 `Customers` 表按右鍵 → **「設計 (Design)」** 
2. 在設計畫面的空白處按右鍵 
3. 選擇 **「關聯性 (Relationships)」** 
4. 左側清單會列出所有與這張表有關的 FK 

> **注意：** 這個對話框同時列出「這張表本身定義的 FK」以及「其他表 FK 指向這張表」，設計目的主要是用來**編輯／新增 FK**，而不是單純查詢。
> 如果關聯很多，需要一筆一筆點開確認，比較費時。

## 三、設定唯一鍵（Unique Key）

### 操作步驟

1. 在資料表設計介面按右鍵，選擇「**索引/索引鍵 (Indexes/Keys)**」。
2. 點擊「**加入 (Add)**」。
3. 在右側屬性欄位中，將「**類型 (Type)**」從預設的「索引」改為「**唯一鍵 (Unique Key)**」。
4. 在「**資料行**」欄位選擇要限制唯一的欄位。
5. 給它一個有意義的名稱（例如 `UQ_Employees_Email`）。
6. 按關閉並儲存。

---
## 四、設定檢查約束（Check Constraint）

### 操作步驟

1. 在目標資料表上按右鍵，選擇「**設計 (Design)**」。
2. 按右鍵，選擇「**檢查條件約束 (Check Constraints)**」。
3. 點選「**加入 (Add)**」。
4. 在右側屬性的「**運算式 (Expression)**」欄位中，輸入你的規則，例如：

```sql
Price > 0
Age >= 0 AND Age <= 150
Status IN ('A', 'B', 'C')
```

5. 按下「關閉」並儲存。

---
## 五、設定預設值（Default Constraint）

預設值的設定位置又跟 Check 不一樣——它直接在欄位的屬性裡設定，不是在獨立的對話框。

### 操作步驟

1. 進入資料表的「**設計 (Design)**」畫面。
2. 點選你要設定預設值的欄位。
3. 看向畫面**最下方**的「**資料行屬性 (Column Properties)**」視窗。
4. 找到「**預設值或繫結 (Default Value or Binding)**」，在旁邊填入預設值。

### 各種資料型別的預設值寫法

|資料型別|寫法範例|說明|
|---|---|---|
|字串|`'未命名'`|記得加單引號|
|數字|`0`|直接填數字|
|布林（bit）|`1` 或 `0`|1 = true，0 = false|
|當下時間|`GETDATE()`|記錄建立當下的時間|
|全域唯一識別碼|`NEWID()`|自動產生 UUID|

存檔後，你可能會發現 SSMS 自動把你輸入的 `0` 變成 `((0))`、`GETDATE()` 變成 `(getdate())`——別緊張，這是正常的語法標準化行為。

## 六、建立索引（Index）

索引是查詢效能的命脈，但記得我們第 1 篇講過——**不要每個欄位都建，要挑常出現在 WHERE 條件裡的欄位。**

### 操作步驟

1. 在「物件總管」中展開目標資料表，找到「**索引 (Indexes)**」資料夾。
2. 在資料夾上按右鍵，選擇「**新增索引 (New Index)**」→ 通常選「**非叢集索引 (Non-Clustered Index)**」（因為叢集索引通常已被主鍵佔用）。
3. 在跳出的視窗中給索引一個有意義的命名，建議格式是 `IX_資料表名稱_欄位名稱`，例如 `IX_Orders_OrderDate`。
4. 點選「**加入 (Add)**」，勾選你想要加速搜尋的欄位。
5. 按下確定即可建立。

如果你發現「新增索引」變成灰色不能點，原因可能如下：

**原因 1（最常見）：設計視窗還開著**。如果你**正開著該表的「設計 (Design)」視窗**，SQL Server 為了防止架構衝突，會鎖定某些右鍵選單。解法：先關閉設計視窗（記得存檔），再去「索引」資料夾右鍵試試看。實務上 90% 的情況都是這個原因，先關設計視窗幾乎一定能解決。

**原因 2：權限不足**。如果你登入的帳號只有 `db_datareader` 權限，沒有 `db_ddladmin` 或 `db_owner` 權限，就無法建立索引。解法：找 DBA 或主管申請適當的權限。記得問清楚開發環境跟正式環境的差異。

## 七、管理統計資料（Statistics）

通常你**不需要手動管理統計資料**，因為 SQL Server 會在你建立索引或執行查詢時自動建立和更新。但有兩種情況你會需要手動介入：

### 情況 1：手動建立統計資料

在「物件總管」中展開目標資料表，找到「**統計資料 (Statistics)**」資料夾，在資料夾上按右鍵，選擇「**新增統計資料 (New Statistics)**」，點選「加入」後選擇要進行統計分析的欄位，按下確定即可。

### 情況 2：更新統計資料（更常用）

當你發現查詢突然變慢，又找不出 SQL 或索引的問題時，很可能是統計資料過期了。有兩種更新方式：

**介面操作**：在「統計資料」資料夾上按右鍵，選擇「**更新統計資料**」；或者在特定的統計項目上按右鍵執行。

**T-SQL 語法（更常用）**：開一個新的 Query 視窗，執行：

```sql
-- 針對單一資料表更新所有統計資料
UPDATE STATISTICS [資料表名稱];

-- 針對整個資料庫更新所有統計資料（火力比較猛）
EXEC sp_updatestats;
```

### 怎麼知道統計資料是不是太舊了？

在「統計資料」資料夾底下的特定項目按右鍵 → 選「**屬性**」，可以看到「**最後更新時間**」。如果上次更新是很久以前，但這段期間資料變動量很大，那就是該更新的時候了。

## 八、操作流程速查表

最後給你一張速查表，未來不確定要去哪裡點時，翻這張就好：

|想設定的東西|操作起點|路徑|
|---|---|---|
|主鍵（PK）|資料表「設計」|點欄位左側方塊 → 右鍵 → 設定主索引鍵|
|外部索引鍵（FK）|資料表「設計」|右鍵 → 關聯性 → 加入|
|唯一鍵（UK）|資料表「設計」|右鍵 → 索引/索引鍵 → 加入（類型選 Unique Key）|
|檢查約束（Check）|資料表「設計」|右鍵 → 檢查條件約束 → 加入|
|預設值（Default）|資料表「設計」|點欄位 → 下方屬性 → 預設值或繫結|
|索引（Index）|索引資料夾|右鍵 → 新增索引 → 非叢集索引|
|更新統計資料|統計資料資料夾或 T-SQL|右鍵更新 / `UPDATE STATISTICS`|
